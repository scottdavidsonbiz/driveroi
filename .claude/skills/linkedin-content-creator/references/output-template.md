# Output Template

## Standard Output Format

When generating content from inputs, use this structure:

---

## Takeaways Extracted

### Takeaway 1: [Title]

**Insight:** [The core learning in one sentence]

**Evidence from input:** "[Direct quote or specific data point]"

**Content angle:** [How this becomes a post - observation, test result, framework, etc.]

---

### Takeaway 2: [Title]

**Insight:** [The core learning]

**Evidence from input:** "[Quote or data]"

**Content angle:** [Post direction]

---

[Repeat for 3-5 takeaways]

---

## Generated Posts

### Post 1: [Short title for reference]

```
[The actual post content]

[Formatted exactly as it would appear on LinkedIn]

[No CTA unless specifically requested]
```

**Status:** Draft - needs review
**Based on:** Takeaway [#]
**Notes:** [Any context about this post - what data it uses, any assumptions, etc.]

---

### Post 2: [Short title]

```
[Post content]
```

**Status:** Draft - needs review
**Based on:** Takeaway [#]
**Notes:** [Context]

---

[Repeat for each post]

---

## Questions / Clarifications Needed

[List any questions that would strengthen the content]

- [Question about missing data?]
- [Clarification on specific claim?]
- [Request for additional context?]

---

## Notes

- All statistics sourced from: [Reference the input]
- Target audience for these posts: [Specify]
- Any posts that need additional data: [List]

---

**Generated:** [Date]
**Input source:** [What was provided]

---

## Append Format

When adding to existing `linkedin-content-drafts.md`:

```markdown
---

## Post [N+1]: [Title]

\`\`\`
[Post content]
\`\`\`

**Status:** Draft - needs review
**Source:** [What input this came from]

---
```
