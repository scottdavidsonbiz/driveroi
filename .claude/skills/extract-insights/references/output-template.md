# Extract Insights Output Template

## File Location

Save to: `content/insights/YYYY-MM-DD-[topic].md`

## Template

```markdown
# Insights: [Transcript Title]
Source: [[transcripts/YYYY-MM-DD-topic]]
Extracted: YYYY-MM-DD

## Pain Points
1. [Specific pain point] - Quote: "[exact quote]"
2. [Another pain point] - Quote: "[exact quote]"

## Buying Triggers
1. [Trigger] - Evidence: "[why this matters now]"

## Objections
1. [Concern raised] - Context: "[when/why mentioned]"

## Opportunities
1. [What we could do] - Because: "[reasoning]"

## Content Ideas
- [Possible LinkedIn post angle]
- [Possible article topic]

## ICP Updates Suggested
- [If this reveals something about ideal customer]
```

## Quality Criteria

**Good Pain Point:** "We lose 30% of calls after 5pm"
**Bad Pain Point:** "Communication is hard"

Pain points must be specific, not generic.
